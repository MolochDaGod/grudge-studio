API Documentation

RESTful API endpoints for the GRUDGE Warlords backend

🔐 Authentication Endpoints

Base URL: auth-gateway-flax.vercel.app — Vercel serverless file-based routing (no /auth/ prefix)

POST /api/login

Description: Login with username and password

Body: { username: string, password: string }

Response: User object + session token

POST /api/register

Description: Create new user account

Body: { username: string, password: string, email?: string }

Response: Created user object

POST /api/connect-wallet

Description: Login/register with Solana wallet

Body: { walletAddress: string, signature: string }

Response: User object + session token

POST /api/guest

Description: Create guest session

Body: None

Response: Guest user object + session token

POST /api/puter

Description: Puter SSO authentication

Body: { puterToken: string }

Response: User object + session token

GET /api/verify

Description: Verify session token (Bearer auth)

Headers: Authorization: Bearer <token>

Response: User object if valid

GET /api/discord

Description: Discord OAuth - returns URL or handles callback with code

Query: ?code=...&state=...

Response: OAuth URL (no code) or redirect with token (with code)

GET /api/github

Description: GitHub OAuth via gruda-projects App - returns URL or handles callback

Query: ?code=...&state=...

Response: OAuth URL (no code) or redirect with token (with code)

POST /api/github

Description: Link GitHub account to existing user

Body: { code: string, userId: string }

Response: { githubId, githubUsername, avatarUrl }

GET /api/health

Description: Auth gateway health check + feature list

Response: { status, version, features[], endpoints{} }

🐙 GitHub App Endpoints (API Server)

Base URL: /api/v1/github — Uses gruda-projects installation tokens for @GrudgeDaDev repos

GET /api/v1/github/status

Description: GitHub App connection status

Response: { configured, appAuthenticated, installationId, owner, appId }

GET /api/v1/github/repos

Description: List GrudgeDaDev repositories

Query: ?page=1&per_page=30&sort=updated

Response: Array of repo objects (name, language, stars, topics)

GET /api/v1/github/repos/:repo

Description: Get specific repo details

Response: Repo object with full metadata

GET /api/v1/github/repos/:repo/contents

Description: Browse repo file tree

Query: ?path=src&ref=main

Response: Array of file/dir entries or file content

GET /api/v1/github/repos/:repo/issues

Description: List repo issues

Query: ?state=open&page=1

Response: Array of issue objects

POST /api/v1/github/repos/:repo/issues

Description: Create issue on a repo

Body: { title: string, body?: string, labels?: string[] }

Response: Created issue with number and URL

POST /api/v1/github/repos/:repo/deploy

Description: Trigger GitHub Actions workflow dispatch

Body: { workflow?: string, ref?: string, inputs?: object }

Response: Success confirmation

GET /api/v1/github/repos/:repo/runs

Description: List recent workflow runs

Response: Array of workflow run objects (status, conclusion, branch)

📦 Game Data Endpoints (API Server)

Base URL: /api/v1/game-data — ObjectStore JSON datasets served via REST

GET /api/v1/game-data

Description: List all available ObjectStore datasets

Response: { datasets: [{name, endpoint}], total, cache }

GET /api/v1/game-data/:dataset

Description: Fetch a specific dataset (e.g., weapons, armor, materials)

Response: { dataset, data, _meta: {source, cachedAge} }

GET /api/v1/game-data/search

Description: Cross-resource search across all datasets

Query: ?q=sword&type=weapons&limit=50

Response: { query, type, count, results[] }

POST /api/v1/game-data/cache/clear

Description: Force clear the ObjectStore cache

Response: { success: true }

🎨 Asset Endpoints (API Server)

Base URL: /api/v1/assets — CRUD for game assets (models, textures, audio)

GET /api/v1/assets

Description: List all assets with filtering

Query: ?type=model&project=default&search=sword&limit=100&offset=0

Response: { assets[], total, limit, offset }

GET /api/v1/assets/:id

Description: Get a single asset by ID

Response: Asset object

POST /api/v1/assets

Description: Create/upload a new asset

Body: { name: string, type?: string, data?: string, project?: string, metadata?: object }

Response: Created asset object (201)

PUT /api/v1/assets/:id

Description: Update an existing asset

Body: { name?, type?, project?, metadata?, synced? }

Response: Updated asset object

DELETE /api/v1/assets/:id

Description: Delete an asset

Response: { success: true, id }

Authentication

Session-Based Auth

All endpoints (except auth endpoints) require authentication.

Session cookie is automatically sent with requests.

Session expires after 7 days of inactivity.

Rate Limiting

API endpoints are rate-limited to prevent abuse.

Standard limit: 100 requests per minute per user.

Burst limit: 20 requests per second.

Error Responses

400 Bad Request - Invalid parameters

401 Unauthorized - Not authenticated

403 Forbidden - Insufficient permissions

404 Not Found - Resource not found

500 Internal Server Error - Server error
