Component Details

🔐 Auth Gateway (Vercel)

  • URL: auth-gateway-flax.vercel.app
  • Runtime: Vercel Serverless Functions
  • Language: JavaScript (CommonJS)
  • DB Driver: postgres.js (raw SQL, no ORM)
  • Auth: bcryptjs password hashing
  • Routing: File-based — each api/*.js is an endpoint

🚀 API Server

  • Framework: Express 4.18
  • Language: JavaScript (ESM modules)
  • Node.js: ≥18.0.0
  • Data Source: ObjectStore via GitHub Pages CDN
  • Caching: In-memory with stale-while-revalidate
  • Status: Local dev on port 3001 (not yet production-deployed)

🗄️ Database (Neon PostgreSQL)

  • Region: us-east-1 (AWS)
  • Schema: Single public schema
  • Live tables: users, accounts, auth_tokens, linked_accounts
  • Driver: postgres.js with raw SQL queries
  • Migrations: Raw ALTER TABLE via /api/migrate endpoint
  • Pooling: max=1 connection per serverless invocation

📦 ObjectStore (GitHub Pages)

  • URL: GrudgeDaDev.github.io/ObjectStore
  • Content: Game data JSON — weapons, armor, materials, etc.
  • Icons: icon-index.json + weapon/armor sprite PNGs
  • Served via: API Server at /api/v1/game-data
  • Browser cache: 5-minute max-age

🔑 Authentication

  • Methods: Username/Password, Wallet, Puter SSO, Discord, GitHub, Guest
  • Wallet: Solana via Crossmint MPC (auto-created on register)
  • Discord: OAuth2 callback at /api/discord
  • GitHub: OAuth via gruda-projects App (Client ID: Iv23li...)
  • Sessions: 7-day random hex tokens stored in auth_tokens table
  • Linking: Multiple providers per user via linked_accounts table

🐙 GitHub App (gruda-projects)

  • Owner: @GrudgeDaDev
  • App ID: 1477428
  • Auth: RS256 JWT → installation tokens (1hr cache)
  • Capabilities: Repo listing, file browsing, issues, workflow dispatch
  • Fallback: PAT (GRUDGEDADEV_GITHUB_TOKEN)
  • Served via: API Server at /api/v1/github

☁️ Puter Sites

  • Account: MolochDaDev
  • grudge-admin.puter.site: This admin dashboard
  • grudachain-ve8e8.puter.site: ObjectStore browser app
  • assets-i4mhsuoyr1.puter.site: Sprite showcase
  • Deploys: puter-cli site deploy

🌐 External Services

  • Crossmint: Solana MPC wallet auto-creation on registration
  • Puter AI: Free Claude/GPT via Puter API (AI agent queries)
  • Vercel: Auth gateway hosting, auto-deploy from GitHub
  • GitHub Pages: ObjectStore CDN for game data JSON

Data Flow Patterns

🎮 Player Login Flow

sequenceDiagram participant Client participant AuthGW as Auth Gateway participant DB as Neon PostgreSQL Client->>AuthGW: POST /api/login {username, password} AuthGW->>DB: SELECT from users WHERE username = ? DB-->>AuthGW: User row with password_hash AuthGW->>AuthGW: bcrypt.compare(password, hash) AuthGW->>DB: INSERT INTO auth_tokens (7-day hex token) AuthGW->>DB: UPDATE users SET last_login_at AuthGW->>DB: SELECT from accounts WHERE user_id AuthGW-->>Client: {token, userId, grudgeId, gold, gbuxBalance}

🐙 GitHub OAuth Flow

sequenceDiagram participant Client participant AuthGW as Auth Gateway participant GH as GitHub API participant DB as Neon PostgreSQL Client->>AuthGW: GET /api/github AuthGW-->>Client: {url: github.com/login/oauth/authorize} Client->>GH: User authorizes app GH-->>AuthGW: GET /api/github?code=ABC AuthGW->>GH: POST /login/oauth/access_token GH-->>AuthGW: access_token AuthGW->>GH: GET /user + GET /user/emails GH-->>AuthGW: GitHub profile + email AuthGW->>DB: Find or create user + linked_accounts upsert AuthGW->>DB: INSERT INTO auth_tokens AuthGW-->>Client: Redirect with token in URL
