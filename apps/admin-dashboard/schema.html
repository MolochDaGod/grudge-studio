Database Schema

Neon PostgreSQL — all tables in public schema

Live Tables

The database uses a single public schema with 4 tables currently in production. The auth-gateway connects via postgres.js with raw SQL queries (no ORM).

👤 users

User accounts — all auth methods converge here

  • id — UUID primary key
  • username — unique, NOT NULL
  • password_hash — bcryptjs hash
  • email — nullable
  • display_name
  • grudge_id — GRUDGE-{timestamp}-{hex} format
  • wallet_address — Solana address (Crossmint)
  • puter_id — Puter SSO ID
  • discord_id — VARCHAR(100), indexed
  • github_id — VARCHAR(100), indexed
  • google_id — VARCHAR(100), indexed
  • avatar_url
  • is_premium — BOOLEAN default false
  • premium_until — BIGINT timestamp
  • last_login_at — BIGINT ms epoch
  • created_at — BIGINT ms epoch

💰 accounts

Economy data — one account per user

  • id — UUID primary key
  • user_id — references users.id
  • display_name
  • gold — starting balance: 1000
  • premium_currency — default 0
  • gbux_balance — default 0
  • account_xp — default 0
  • wallet_address
  • wallet_type
  • created_at — BIGINT ms epoch

🔑 auth_tokens

Session tokens — 7-day expiry

  • id — UUID primary key
  • user_id — references users.id
  • token — 32-byte random hex string
  • token_type — 'standard', 'discord', 'github', 'puter', 'guest'
  • expires_at — BIGINT (Date.now() + 7 days)
  • created_at — BIGINT ms epoch

🔗 linked_accounts

OAuth provider links — multiple providers per user

  • id — UUID primary key (gen_random_uuid)
  • user_id — UUID NOT NULL
  • provider — VARCHAR(30) ('discord', 'github')
  • provider_id — VARCHAR(100) NOT NULL
  • provider_email — VARCHAR(255)
  • provider_username — VARCHAR(100)
  • provider_avatar — TEXT
  • access_token — TEXT
  • refresh_token — TEXT
  • token_expires_at — BIGINT
  • linked_at — BIGINT NOT NULL
  • last_used_at — BIGINT
  • UNIQUE(provider, provider_id)
  • UNIQUE(user_id, provider)

Actual CREATE Statements

Derived from auth-gateway code (login.js, register.js, verify.js, migrate.js, github.js, discord.js)

users table (columns confirmed from SELECT/INSERT queries)

-- Core columns (used in register.js INSERT)
id              UUID PRIMARY KEY
username        TEXT UNIQUE NOT NULL
password_hash   TEXT NOT NULL
email           TEXT
display_name    TEXT
grudge_id       TEXT              -- GRUDGE-{timestamp}-{hex}
wallet_address  TEXT              -- Solana (set by Crossmint on register)
puter_id        TEXT
is_premium      BOOLEAN DEFAULT false
premium_until   BIGINT
last_login_at   BIGINT
created_at      BIGINT

-- OAuth columns (added via migrate.js ALTER TABLE)
discord_id      VARCHAR(100) UNIQUE
github_id       VARCHAR(100) UNIQUE
google_id       VARCHAR(100) UNIQUE
avatar_url      TEXT

-- Indexes
idx_users_discord  ON users(discord_id)
idx_users_github   ON users(github_id)
idx_users_google   ON users(google_id)

accounts table (from register.js INSERT)

id                UUID PRIMARY KEY
user_id           UUID NOT NULL          -- FK to users.id
display_name      TEXT
gold              INTEGER DEFAULT 1000   -- starting gold
premium_currency  INTEGER DEFAULT 0
gbux_balance      INTEGER DEFAULT 0
account_xp        INTEGER DEFAULT 0
wallet_address    TEXT                   -- also on accounts (verify.js reads it)
wallet_type       TEXT
created_at        BIGINT

auth_tokens table (from login.js INSERT)

id          UUID PRIMARY KEY
user_id     UUID NOT NULL
token       TEXT NOT NULL          -- crypto.randomBytes(32).toString('hex')
token_type  VARCHAR                -- 'standard', 'discord', 'github', 'puter', 'guest'
expires_at  BIGINT NOT NULL        -- Date.now() + 7 days in ms
created_at  BIGINT NOT NULL

linked_accounts table (from migrate.js CREATE TABLE)

CREATE TABLE IF NOT EXISTS linked_accounts (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id           UUID NOT NULL,
  provider          VARCHAR(30) NOT NULL,
  provider_id       VARCHAR(100) NOT NULL,
  provider_email    VARCHAR(255),
  provider_username VARCHAR(100),
  provider_avatar   TEXT,
  access_token      TEXT,
  refresh_token     TEXT,
  token_expires_at  BIGINT,
  linked_at         BIGINT NOT NULL,
  last_used_at      BIGINT,
  UNIQUE(provider, provider_id),
  UNIQUE(user_id, provider)
);

-- Indexes
CREATE INDEX idx_linked_provider ON linked_accounts(provider, provider_id);
CREATE INDEX idx_linked_user ON linked_accounts(user_id);

Table Relationships

users (PK: id)
├── username, password_hash, email, display_name
├── grudge_id — canonical Grudge Studio identifier
├── wallet_address — Solana (Crossmint MPC)
├── puter_id, discord_id, github_id, google_id — provider IDs
├── avatar_url, is_premium, premium_until
└── last_login_at, created_at

accounts (PK: id, FK: user_id → users.id)
├── display_name — "{username}'s Account"
├── gold (1000), premium_currency (0), gbux_balance (0), account_xp (0)
└── wallet_address, wallet_type

auth_tokens (PK: id, FK: user_id → users.id)
├── token — 64-char hex string, checked by /api/verify
├── token_type — standard | discord | github | puter | guest
└── expires_at — 7 days from creation

linked_accounts (PK: id, FK: user_id → users.id)
├── provider + provider_id — UNIQUE (one link per provider globally)
├── user_id + provider — UNIQUE (one provider per user)
├── provider_email, provider_username, provider_avatar
├── access_token, refresh_token, token_expires_at
└── linked_at, last_used_at — ON CONFLICT DO UPDATE

Database Conventions

Naming

snake_case columns (password_hash, display_name, etc.)

UUID primary keys via gen_random_uuid()

Timestamps

BIGINT milliseconds since epoch (Date.now())

NOT Postgres TIMESTAMP — allows JS-native handling

Driver

postgres.js (raw tagged template SQL)

No ORM — all queries are explicit

Migrations

ALTER TABLE via /api/migrate endpoint

Protected by MIGRATION_SECRET or ?setup= param
