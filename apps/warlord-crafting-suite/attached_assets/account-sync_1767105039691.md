# Account Sync with SHA Change Detection

Efficient account synchronization system using SHA-256 hashing and batch updates.

## Overview

The account sync system provides:
1. **SHA-based change detection** - Skip writes when data hasn't changed
2. **Batch update queue** - Coalesce changes before writing to Google Sheets
3. **Activity logging** - Track inventory/equipment changes separately

## Architecture

```
User Action → Backend Queue → SHA Check → Batch Write → Google Sheets
                    ↓
            Activity Buffer
                    ↓
            Puter KV/FS (per-user logs)
```

## Data Storage Strategy

| Data Type | Storage | Update Frequency | Reason |
|-----------|---------|------------------|--------|
| Account profiles | Google Sheet | On change (SHA check) | Low churn, auditable |
| Inventory changes | Puter KV queue | Immediate | High churn, per-user |
| Equipment acquired | Puter FS logs | Append-only | Historical record |
| Activity summary | Backend buffer | In-memory | Real-time display |

## SHA Change Detection

### How It Works

1. When account data is queued for update, we generate a SHA-256 hash
2. Hash is compared against the stored hash for that account
3. If hashes match, the update is skipped (no changes)
4. If hashes differ, update is queued for batch write

### Hash Generation

The system maintains a **full account snapshot** for each account and generates hashes from the complete normalized data:

```javascript
function generateSHA256(data) {
  // Normalize all fields to ensure consistent hashing
  const normalized = {
    id: data.id || '',
    username: data.username || '',
    email: data.email || '',
    displayName: data.displayName || '',
    puterId: data.puterId || '',
    avatarUrl: data.avatarUrl || '',
    isPremium: data.isPremium || false,
    premiumUntil: data.premiumUntil || '',
    createdAt: data.createdAt || '',
    lastLoginAt: data.lastLoginAt || '',
    settings: data.settings || {},
  };
  // Keys are sorted for consistent ordering
  const json = JSON.stringify(normalized, Object.keys(normalized).sort());
  return crypto.createHash('sha256')
    .update(json)
    .digest('hex')
    .substring(0, 16); // First 16 chars for compactness
}
```

When a partial update is received, it's merged with the existing snapshot before hashing, ensuring consistent comparison.

### Benefits

- **90%+ reduction** in redundant Google Sheets writes
- **Faster response times** - No waiting for API calls on unchanged data
- **Rate limit friendly** - Stay well under Google's 60 writes/minute

## Batch Update Queue

### Configuration

```javascript
const BATCH_DELAY_MS = 30000;  // 30 seconds
const MAX_BATCH_SIZE = 50;     // Max accounts per batch
```

### Overflow Handling

If more than 50 accounts have pending changes, the flush processes 50 and reschedules automatically for the remaining. **No data is lost** - overflow entries stay in the queue.

### Coalescing

Multiple updates to the same account within the batch window are merged:

```
Update 1: { displayName: "John" }
Update 2: { lastLoginAt: "2025-12-29" }
Update 3: { displayName: "Johnny" }

Result: { displayName: "Johnny", lastLoginAt: "2025-12-29" }
```

Only the final merged state is written to Google Sheets.

## Puter Activity Logging (Client-Side)

For high-frequency events, log directly to Puter storage:

```javascript
// Include in your Puter app
const script = await fetch('/api/sync/activity-client.js?accountId=YOUR_ID');
eval(await script.text());

// Log activity
await logActivityToPuter('item_acquired', {
  itemId: 'sword-123',
  itemName: 'Iron Sword',
  tier: 3
});

// Read activity
const recent = await getActivityFromPuter(50);
```


## Best Practices

1. **Use SHA check for all account updates** - Never write directly to sheets
2. **Batch high-frequency updates** - Let the queue coalesce changes
3. **Log activity to Puter for speed** - Inventory changes go to KV/FS
4. **Summarize daily** - Aggregate activity logs to sheets once per day
5. **Handle failures gracefully** - Retry failed writes on next batch

## Integration Example

```typescript
// In your route handler
import { queueAccountChange, logActivity } from './accountSync';

// Update account (SHA-checked, batched)
app.post('/api/profile/update', async (req, res) => {
  const result = queueAccountChange(req.body.accountId, {
    displayName: req.body.displayName,
    avatarUrl: req.body.avatarUrl
  });
  res.json(result);
});

// Log item acquisition (high-frequency)
app.post('/api/items/acquire', async (req, res) => {
  // Save to database
  await storage.addInventoryItem(...);
  
  // Log activity (in-memory buffer)
  logActivity(req.body.accountId, 'item_acquired', {
    itemId: req.body.itemId,
    itemName: req.body.itemName
  });
  
  res.json({ success: true });
});
```
