# Puter Apps API

Create and manage Puter applications programmatically.

## Overview

The Apps API allows you to:
- Create and publish apps
- Update app metadata
- Manage app settings
- Handle file type associations

## App Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `name` | string | Unique app identifier |
| `indexURL` | string | URL to app's main page |
| `title` | string | Display name |
| `description` | string | User-facing description |
| `icon` | string | App icon URL |
| `maximizeOnStart` | boolean | Start maximized |
| `background` | boolean | Run in background |
| `filetypeAssociations` | array | File types app can open |
| `metadata` | object | Custom key-value data |

## Core Functions

### Create App
```javascript
const app = await puter.apps.create('grudge-crafting', 'https://grudge-crafting.puter.site');
console.log(`App created: ${app.name}`);
```

### Update App
```javascript
const updated = await puter.apps.update('grudge-crafting', {
    title: 'GRUDGE Warlords Crafting',
    description: 'Craft weapons, armor, and potions for your adventure!',
    icon: 'https://example.com/icon.png',
    maximizeOnStart: true,
    metadata: {
        version: '2.4.0',
        author: 'Grudge Studio'
    }
});
```

### Get App Info
```javascript
const app = await puter.apps.get('grudge-crafting');
console.log(`Title: ${app.title}`);
console.log(`Version: ${app.metadata?.version}`);
```

### List Apps
```javascript
const apps = await puter.apps.list();
apps.forEach(app => {
    console.log(`${app.name}: ${app.title}`);
});
```

### Delete App
```javascript
await puter.apps.delete('old-app-name');
```

## File Type Associations

```javascript
// Register app to open certain file types
await puter.apps.update('grudge-crafting', {
    filetypeAssociations: [
        '.grudge',           // Custom extension
        '.json',             // Config files
        'application/json'   // MIME type
    ]
});
```

## GRUDGE App Configuration

### Full App Setup
```javascript
// Configuration - set these values directly or via build-time substitution
const APP_CONFIG = {
    appName: 'grudge-warlords',
    indexUrl: 'https://grudge-warlords.puter.site', // Your Puter app URL
    version: '2.5.0'
};

async function setupGrudgeApp() {
    const { appName, indexUrl, version } = APP_CONFIG;
    
    try {
        // Check if exists
        await puter.apps.get(appName);
        
        // Update existing
        await puter.apps.update(appName, {
            title: 'GRUDGE Warlords',
            description: 'Epic crafting and character progression system',
            indexURL: indexUrl,
            maximizeOnStart: true,
            filetypeAssociations: ['.grudge', '.grudgechar'],
            metadata: {
                version: version,
                features: ['crafting', 'professions', 'combat'],
                backend: 'replit'
            }
        });
        
    } catch (e) {
        // Create new
        await puter.apps.create(appName, indexUrl);
        await puter.apps.update(appName, {
            title: 'GRUDGE Warlords',
            description: 'Epic crafting and character progression system'
        });
    }
}
```

### Environment-Based Configuration
```javascript
// For Puter static apps, configure URLs directly or via KV storage
// Node.js/build scripts can use process.env; browser code cannot

const config = {
    development: {
        name: 'grudge-dev',
        indexURL: 'http://localhost:5000'
    },
    staging: {
        name: 'grudge-staging',
        indexURL: 'https://grudge-staging.puter.site'
    },
    production: {
        name: 'grudge-warlords',
        indexURL: 'https://grudge-warlords.puter.site'
    }
};

async function deployToEnvironment(env) {
    const { name, indexURL } = config[env];
    
    try {
        await puter.apps.get(name);
        await puter.apps.update(name, { indexURL });
    } catch (e) {
        await puter.apps.create(name, indexURL);
    }
    
    return name;
}
```

## Best Practices

1. **Use unique names**: App names must be unique per user
2. **Meaningful metadata**: Store version, features, and config
3. **Handle existing apps**: Check before create, update if exists
4. **Clean file associations**: Only register types your app handles
5. **Update atomically**: Batch attribute changes in single update
