# Puter Hosting API

Deploy and manage websites on Puter's infrastructure.

## Overview

The Hosting API enables programmatic website deployment. Perfect for:
- Static site generators
- Website builders
- Deployment automation
- Multi-tenant platforms

## Core Functions

### Create Hosting
```javascript
// Create empty site
const site = await puter.hosting.create('my-subdomain');
console.log(`Site: https://${site.subdomain}.puter.site`);

// Create site with directory
await puter.fs.mkdir('my-website');
await puter.fs.write('my-website/index.html', '<h1>Hello World</h1>');
const site = await puter.hosting.create('my-subdomain', 'my-website');
```

### List All Sites
```javascript
const sites = await puter.hosting.list();
sites.forEach(site => {
    console.log(`${site.subdomain}.puter.site - ${site.uid}`);
});
```

### Get Site Info
```javascript
const site = await puter.hosting.get('my-subdomain');
console.log(`Subdomain: ${site.subdomain}`);
console.log(`UID: ${site.uid}`);
console.log(`Root Directory: ${site.root_dir}`);
```

### Update Site
```javascript
// Point subdomain to new directory
await puter.hosting.update('my-subdomain', 'new-website-directory');
```

### Delete Site
```javascript
await puter.hosting.delete('my-subdomain');
```

## GRUDGE Deployment Examples

### Deploy Auth Service
```javascript
async function deployAuthService() {
    const subdomain = 'grudge-auth';
    const directory = 'puter-app';
    
    // Create directory if not exists
    try {
        await puter.fs.mkdir(directory);
    } catch (e) { /* exists */ }
    
    // Upload files
    await puter.fs.write(`${directory}/index.html`, authHtml);
    await puter.fs.write(`${directory}/sprite-analyzer.html`, spriteHtml);
    
    // Create or update hosting
    try {
        await puter.hosting.get(subdomain);
        await puter.hosting.update(subdomain, directory);
    } catch (e) {
        await puter.hosting.create(subdomain, directory);
    }
    
    return `https://${subdomain}.puter.site`;
}
```

### Deploy Static Game Build
```javascript
async function deployGameBuild(buildDir, version) {
    const subdomain = `grudge-${version.replace(/\./g, '-')}`;
    
    // Upload all build files
    const files = await puter.fs.readdir(buildDir);
    for (const file of files) {
        if (!file.is_dir) {
            const blob = await puter.fs.read(`${buildDir}/${file.name}`);
            await puter.fs.write(`deploy/${file.name}`, blob);
        }
    }
    
    // Create hosting
    const site = await puter.hosting.create(subdomain, 'deploy');
    return site;
}
```

### Multi-Environment Deployment
```javascript
const ENVIRONMENTS = {
    dev: 'grudge-dev',
    staging: 'grudge-staging',
    prod: 'grudge-warlords'
};

async function deployToEnvironment(env, directory) {
    const subdomain = ENVIRONMENTS[env];
    if (!subdomain) throw new Error(`Unknown environment: ${env}`);
    
    try {
        await puter.hosting.update(subdomain, directory);
    } catch (e) {
        await puter.hosting.create(subdomain, directory);
    }
    
    return `https://${subdomain}.puter.site`;
}
```

## Site Structure

A typical GRUDGE Puter site:
```
puter-app/
├── index.html           # Main auth page
├── sprite-analyzer.html # AI sprite tool
├── game/
│   ├── index.html      # Game entry
│   └── assets/         # Game assets
└── docs/
    └── index.html      # Documentation
```

## Best Practices

1. **Use meaningful subdomains**: `grudge-auth`, `grudge-game`, `grudge-tools`
2. **Always have index.html**: Required for root access
3. **Handle existing sites**: Check with `get()` before `create()`
4. **Clean up old deployments**: Delete unused sites to free subdomains
5. **Version deployments**: Use version numbers in subdomains for rollback
