# Puter Serverless Workers API

Run and manage serverless JavaScript functions in the cloud.

## Overview

Puter Workers are serverless functions that run JavaScript in the cloud. Perfect for:
- REST APIs and webhooks
- Data processing pipelines
- Backend services without servers
- Integrating with Puter's cloud services

## Worker Router System

Workers use a router-based system similar to Express.js:

### Basic Routes
```javascript
// GET endpoint
router.get("/api/hello", async ({ request }) => {
    return { message: "Hello, World!" };
});

// POST endpoint
router.post("/api/data", async ({ request }) => {
    const body = await request.json();
    return { received: body, processed: true };
});

// PUT endpoint
router.put("/api/update/:id", async ({ request, params }) => {
    const { id } = params;
    const body = await request.json();
    return { id, updated: body };
});

// DELETE endpoint
router.delete("/api/delete/:id", async ({ request, params }) => {
    return { deleted: params.id };
});
```

### URL Parameters
```javascript
// Capture dynamic segments with :paramName
router.get("/api/characters/:characterId", async ({ request, params }) => {
    const { characterId } = params;
    return { characterId };
});

// Multiple parameters
router.get("/api/users/:userId/items/:itemId", async ({ request, params }) => {
    const { userId, itemId } = params;
    return { userId, itemId };
});
```

### Request Handling
```javascript
router.post("/api/process", async ({ request }) => {
    // Get JSON body
    const body = await request.json();
    
    // Get headers
    const authHeader = request.headers.get('Authorization');
    
    // Get query parameters
    const url = new URL(request.url);
    const page = url.searchParams.get('page') || '1';
    
    return { body, hasAuth: !!authHeader, page };
});
```

### Response Types
```javascript
// Auto JSON response (recommended)
router.get("/api/simple", async () => {
    return { status: "ok", data: [1, 2, 3] };
});

// Custom Response with status code
router.get("/api/error", async () => {
    return new Response(JSON.stringify({ error: "Not found" }), {
        status: 404,
        headers: { "Content-Type": "application/json" }
    });
});

// HTML response
router.get("/api/html", async () => {
    return new Response("<h1>Hello</h1>", {
        headers: { "Content-Type": "text/html" }
    });
});
```

## Puter API Integration in Workers

Workers can access all Puter APIs via the global `puter` object:

### KV Store in Workers
```javascript
router.post("/api/kv/set", async ({ request }) => {
    const { key, value } = await request.json();
    
    if (!key || value === undefined) {
        return new Response(JSON.stringify({ error: "Key and value required" }), {
            status: 400,
            headers: { "Content-Type": "application/json" }
        });
    }
    
    // Use prefix to namespace keys (puter is global in workers)
    await puter.kv.set("grudge_" + key, value);
    return { saved: true, key };
});

router.get("/api/kv/get/:key", async ({ params }) => {
    const value = await puter.kv.get("grudge_" + params.key);
    
    if (!value) {
        return new Response(JSON.stringify({ error: "Key not found" }), {
            status: 404,
            headers: { "Content-Type": "application/json" }
        });
    }
    
    return { key: params.key, value };
});
```

### File System in Workers
```javascript
router.post("/api/files/save", async ({ request }) => {
    const { filename, content } = await request.json();
    
    // puter is available as a global in the worker runtime
    await puter.fs.write(`data/${filename}`, content, {
        createMissingParents: true
    });
    
    return { saved: true, filename };
});

router.get("/api/files/read/:filename", async ({ params }) => {
    try {
        const blob = await puter.fs.read(`data/${params.filename}`);
        const content = await blob.text();
        return { filename: params.filename, content };
    } catch (e) {
        return new Response(JSON.stringify({ error: "File not found" }), {
            status: 404,
            headers: { "Content-Type": "application/json" }
        });
    }
});
```

### AI in Workers
```javascript
router.post("/api/ai/chat", async ({ request }) => {
    const { message } = await request.json();
    
    // puter.ai is available globally in workers
    const response = await puter.ai.chat(message);
    
    return { response };
});
```

## Workers Management API

### Create Worker
```javascript
const workerCode = `
router.get("/api/hello", async () => {
    return { message: "Hello from GRUDGE Worker!" };
});
`;

const worker = await puter.workers.create('grudge-api', workerCode);
console.log(`Worker URL: ${worker.url}`);
```

### List Workers
```javascript
const workers = await puter.workers.list();
workers.forEach(w => {
    console.log(`${w.name}: ${w.url}`);
});
```

### Get Worker Info
```javascript
const worker = await puter.workers.get('grudge-api');
console.log(worker);
```

### Execute Worker
```javascript
// Execute GET request
const result = await puter.workers.exec('grudge-api', '/api/hello');

// Execute POST request
const result = await puter.workers.exec('grudge-api', '/api/data', {
    method: 'POST',
    body: JSON.stringify({ key: 'value' }),
    headers: { 'Content-Type': 'application/json' }
});
```

### Delete Worker
```javascript
await puter.workers.delete('grudge-api');
```

## GRUDGE Worker Examples

### Character API Worker
```javascript
// Character management worker
router.get("/api/characters/:id", async ({ params }) => {
    const data = await me.puter.kv.get(`character_${params.id}`);
    if (!data) {
        return new Response(JSON.stringify({ error: "Character not found" }), {
            status: 404,
            headers: { "Content-Type": "application/json" }
        });
    }
    return JSON.parse(data);
});

router.post("/api/characters", async ({ request }) => {
    const character = await request.json();
    const id = Date.now().toString(36);
    await me.puter.kv.set(`character_${id}`, JSON.stringify(character));
    return { id, ...character };
});

router.put("/api/characters/:id/gold", async ({ request, params }) => {
    const { amount } = await request.json();
    const data = await me.puter.kv.get(`character_${params.id}`);
    if (!data) {
        return new Response(JSON.stringify({ error: "Character not found" }), {
            status: 404,
            headers: { "Content-Type": "application/json" }
        });
    }
    const character = JSON.parse(data);
    character.gold = (character.gold || 0) + amount;
    await me.puter.kv.set(`character_${params.id}`, JSON.stringify(character));
    return character;
});
```

## Best Practices

1. **Always validate input**: Check request body and parameters
2. **Use proper HTTP status codes**: 200, 201, 400, 404, 500
3. **Namespace KV keys**: Prevent collisions between workers
4. **Handle errors gracefully**: Return meaningful error messages
5. **Keep workers focused**: One worker per domain/feature
