## Local Development

### Prerequisites
- Node.js 20+
- pnpm 8+
- PostgreSQL 13+

### Setup

1. **Clone and install**:
```bash
git clone https://github.com/GrudgeDaDev/grudge-studio.git
cd grudge-studio
pnpm install
```

2. **Configure environment**:
```bash
cp .env.example .env.local
# Edit with your database URL
```

3. **Initialize database**:
```bash
pnpm db:push
```

4. **Start development**:
```bash
pnpm dev
# Opens http://localhost:5000
```

## Docker

### Prerequisites
- Docker 20+
- Docker Compose 2+

### Local Deployment

1. **Start with Docker Compose**:
```bash
docker-compose up -d
```

This automatically:
- Sets up PostgreSQL database
- Builds the application
- Starts the app on port 5000
- Configures health checks

2. **Check status**:
```bash
docker-compose ps
docker-compose logs -f app
```

3. **Database operations**:
```bash
# Push schema
docker-compose exec app pnpm db:push

# Open studio
docker-compose exec app pnpm db:studio
```

4. **Stop**:
```bash
docker-compose down
```

### Production Docker Image

Build production image:
```bash
docker build -t grudge-studio:latest .
docker tag grudge-studio:latest grudge-studio:1.0.0
```

Run container:
```bash
docker run -d \
  -p 5000:5000 \
  -e DATABASE_URL="postgresql://user:pass@host/db" \
  -e NODE_ENV=production \
  -e SESSION_SECRET="your-secret-key" \
  grudge-studio:latest
```

Push to registry:
```bash
docker login
docker push grudge-studio:latest
```

## Vercel

Deploy Warlord-Crafting-Suite to Vercel:

### 1. Create Vercel Project

```bash
npm install -g vercel
vercel login
vercel link
```

### 2. Configure Build

In `vercel.json`:
```json
{
  "buildCommand": "cd apps/warlord-crafting-suite && npm run build",
  "outputDirectory": "apps/warlord-crafting-suite/dist",
  "framework": "express",
  "env": {
    "DATABASE_URL": "@database_url",
    "NODE_ENV": "production"
  }
}
```

### 3. Add Environment Variables

In Vercel dashboard:
- `DATABASE_URL` - PostgreSQL connection string
- `SESSION_SECRET` - Secret key
- `ADMIN_PASSWORD` - Admin password
- Other optional vars

### 4. Deploy

```bash
vercel deploy --prod
```

## Railway

Deploy with Railway using their GitHub integration:

### 1. Connect GitHub

1. Go to [railway.app](https://railway.app)
2. Create new project
3. Connect GitHub repository
4. Select `grudge-studio`

### 2. Add PostgreSQL

In Railway dashboard:
- Click "Add Service"
- Select "PostgreSQL"
- Configure database

### 3. Set Environment Variables

- `DATABASE_URL` - Auto-generated by Railway
- `NODE_ENV=production`
- `SESSION_SECRET` - Generate random
- Others from `.env.example`

### 4. Configure Build

Create `railway.toml`:
```toml
[build]
builder = "nixpacks"
buildCommand = "pnpm install && pnpm build"

[deploy]
startCommand = "node apps/warlord-crafting-suite/dist/index.cjs"
healthcheckPath = "/api/health"
healthcheckTimeout = 30
```

### 5. Deploy

Push to GitHub and Railway auto-deploys:
```bash
git push origin main
```

## Replit

Deploy to Replit (if you still use it):

### 1. Import Repository

1. Create new Replit
2. Import from GitHub: `GrudgeDaDev/grudge-studio`

### 2. Configure `.replit`

```bash
run = "pnpm install && pnpm dev"
modules = ["nodejs-20", "postgresql"]
```

### 3. Set Secrets

In Replit Secrets:
- `DATABASE_URL=postgresql://...`
- `SESSION_SECRET=...`
- Others from `.env.example`

### 4. Run

```bash
pnpm install
pnpm db:push
pnpm dev
```

## Environment Variables

### Required
- `DATABASE_URL` - PostgreSQL connection string
- `NODE_ENV` - `development` or `production`
- `SESSION_SECRET` - Random secure string

### Optional
- `ADMIN_PASSWORD` - Admin auth password
- `PORT` - Server port (default 5000)
- `GOOGLE_SHEET_*` - Google Sheets integration
- `PUTER_*` - Puter cloud integration
- `OPENAI_API_KEY` - For AI agents

See `.env.example` for complete list.

## Database Setup

### Local PostgreSQL

```bash
# Install PostgreSQL
# macOS
brew install postgresql@16

# Ubuntu
sudo apt install postgresql-16

# Start service
pg_ctl -D /usr/local/var/postgres start

# Create database
createdb grudge_studio
```

### Push Schema

```bash
export DATABASE_URL="postgresql://user:password@localhost/grudge_studio"
pnpm db:push
```

### Backup Database

```bash
# Backup
pg_dump $DATABASE_URL > backup.sql

# Restore
psql $DATABASE_URL < backup.sql
```

## Troubleshooting

### Build fails with "Cannot find module"

```bash
# Clear cache and reinstall
pnpm clean
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### Database connection error

```bash
# Verify connection string
echo $DATABASE_URL

# Test connection
psql $DATABASE_URL -c "SELECT 1"
```

### Port already in use

```bash
# Change port
PORT=3000 pnpm dev

# Or kill process
lsof -i :5000
kill -9 <PID>
```

### Docker build fails

```bash
# Clear Docker cache
docker builder prune
docker-compose down -v

# Rebuild
docker-compose up --build
```
