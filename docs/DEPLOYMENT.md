## Local Development

### Prerequisites
- Node.js 20+
- pnpm 8+
- PostgreSQL 16+ (local) or Railway `DATABASE_URL`

### Setup

1. **Clone and install**:
```bash
git clone https://github.com/MolochDaGod/grudge-studio.git
cd grudge-studio
pnpm install
```

2. **Configure environment**:
```bash
cp .env.example .env
# Set DATABASE_URL and other vars — see .env.example
```

3. **Initialize database**:
```bash
pnpm db:push
```

4. **Start development**:
```bash
pnpm dev
# Opens http://localhost:5000
```

## Docker

### Prerequisites
- Docker 20+
- Docker Compose 2+

### Local Deployment

1. **Start with Docker Compose**:
```bash
docker-compose up -d
```

This automatically:
- Sets up PostgreSQL database
- Builds the application
- Starts the app on port 5000
- Configures health checks

2. **Check status**:
```bash
docker-compose ps
docker-compose logs -f app
```

3. **Database operations**:
```bash
# Push schema
docker-compose exec app pnpm db:push

# Open studio
docker-compose exec app pnpm db:studio
```

4. **Stop**:
```bash
docker-compose down
```

### Production Docker Image

Build production image:
```bash
docker build -t grudge-studio:latest .
docker tag grudge-studio:latest grudge-studio:1.0.0
```

Run container:
```bash
docker run -d \
  -p 5000:5000 \
  -e DATABASE_URL="postgresql://user:pass@host/db" \
  -e NODE_ENV=production \
  -e SESSION_SECRET="your-secret-key" \
  grudge-studio:latest
```

Push to registry:
```bash
docker login
docker push grudge-studio:latest
```

## Vercel

Deploy Warlord-Crafting-Suite to Vercel:

### 1. Create Vercel Project

```bash
npm install -g vercel
vercel login
vercel link
```

### 2. Configure Build

The `vercel.json` in the repo root is already configured:
```json
{
  "buildCommand": "cd apps/warlord-crafting-suite && npx vite build",
  "outputDirectory": "apps/warlord-crafting-suite/dist/public",
  "framework": "vite",
  "env": {
    "VITE_GAME_API_URL": "https://api.grudge-studio.com",
    "VITE_AUTH_API_URL": "https://id.grudge-studio.com",
    "VITE_OBJECTSTORE_URL": "https://molochdagod.github.io/ObjectStore",
    "NODE_ENV": "production"
  }
}
```

### 3. Add Environment Variables

In Vercel dashboard (use Vercel Secrets with `@` prefix):
- `SESSION_SECRET` — `@session_secret`
- `GRUDGE_AUTH_URL` — `@grudge_auth_url`
- `GRUDGE_API_URL` — `@grudge_api_url`
- See `vercel.json` for the full env mapping

### 4. Deploy

```bash
vercel deploy --prod
```

## Railway

Railway hosts the backend API, PostgreSQL database, and WebSocket server.

### 1. Connect GitHub

1. Go to [railway.app](https://railway.app)
2. Create new project
3. Connect GitHub repository
4. Select `grudge-studio`

### 2. Add PostgreSQL

In Railway dashboard:
- Click "Add Service" → "PostgreSQL"
- Railway auto-generates `DATABASE_URL`

### 3. Set Environment Variables

- `DATABASE_URL` — Auto-generated by Railway
- `NODE_ENV=production`
- `SESSION_SECRET` — Generate random
- `GRUDGE_AUTH_URL=https://id.grudge-studio.com`
- `GRUDGE_API_URL=https://api.grudge-studio.com`
- See `.env.example` for the full list

### 4. Configure Build

Create `railway.toml`:
```toml
[build]
builder = "nixpacks"
buildCommand = "pnpm install && pnpm build"

[deploy]
startCommand = "node apps/warlord-crafting-suite/dist/index.cjs"
healthcheckPath = "/api/health"
healthcheckTimeout = 30
```

### 5. Deploy

Push to GitHub and Railway auto-deploys:
```bash
git push origin master
```

## Environment Variables

### Required
- `DATABASE_URL` — PostgreSQL connection string (Railway auto-generates)
- `NODE_ENV` — `development` or `production`
- `SESSION_SECRET` — Random secure string

### Grudge Backend URLs
- `GRUDGE_API_URL` — `https://api.grudge-studio.com`
- `GRUDGE_AUTH_URL` — `https://id.grudge-studio.com`
- `GRUDGE_ASSETS_URL` — `https://assets.grudge-studio.com`
- `GRUDGE_WS_URL` — `https://ws.grudge-studio.com`

### Vite Frontend Vars (prefixed `VITE_`)
- `VITE_GAME_API_URL` — `https://api.grudge-studio.com`
- `VITE_AUTH_API_URL` — `https://id.grudge-studio.com`
- `VITE_OBJECTSTORE_URL` — `https://molochdagod.github.io/ObjectStore`
- `VITE_WS_URL` — `wss://ws.grudge-studio.com`

### Optional
- `PORT` — Server port (default 5000)
- `PUTER_ENABLED` — Enable Puter.js integration

See `.env.example` for the complete list. Secrets go in the Vercel/Railway dashboard, never in the repo.

## Database Setup

The project uses PostgreSQL with Drizzle ORM. In production, Railway provides the database.

### Local Development

Install PostgreSQL locally, create a database, and set `DATABASE_URL`:

```bash
# Set connection string
$env:DATABASE_URL = "postgresql://user:password@localhost:5432/grudge_studio"

# Push schema
pnpm db:push
```

### Production (Railway)

Railway auto-provisions PostgreSQL and injects `DATABASE_URL`. No manual DB setup needed.

### Backup

```bash
pg_dump $DATABASE_URL > backup.sql
psql $DATABASE_URL < backup.sql
```

## Troubleshooting

### Build fails with "Cannot find module"

```bash
# Clear cache and reinstall
pnpm clean
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### Database connection error

```bash
# Verify connection string
echo $DATABASE_URL

# Test connection
psql $DATABASE_URL -c "SELECT 1"
```

### Port already in use

```bash
# Change port
PORT=3000 pnpm dev

# Or kill process
lsof -i :5000
kill -9 <PID>
```

### Docker build fails

```bash
# Clear Docker cache
docker builder prune
docker-compose down -v

# Rebuild
docker-compose up --build
```
