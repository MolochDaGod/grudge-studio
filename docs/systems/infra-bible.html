Vercel Configuration

VERCEL BEST PRACTICES

How every Vercel project should be configured for maximum reliability, performance, and shared config.

Project Structure

Monorepo with Turborepo

All Grudge apps should live in one monorepo. Turborepo handles build caching. Shared packages (auth, db, ui) live in /packages. Deploy individual apps as separate Vercel projects from the same repo.

Environment Strategy

3-Tier Environments

Production (main branch) → live game
Staging (staging branch) → Crossmint devnet, Grudge Backend (MySQL 8) staging
Preview (every PR) → auto-deployed, isolated

Performance

Edge Runtime for APIs

Use export const runtime = 'edge' for read-heavy endpoints (player profile, leaderboard). Use Node.js runtime for write ops (GBux, minting) that need DB connections.

Security

Middleware Auth Guard

Use Vercel middleware to verify Puter JWT on every API route. Block unauthenticated requests before they hit your API handlers. Never do auth checks inside individual route files.

// middleware.ts — Auth guard for all API routes
import { NextResponse } from 'next/server'
import { verifyJWT } from 'shared/auth'

export async function middleware(req) {
  const path = req.nextUrl.pathname

  // Protect all game API routes
  if (path.startsWith('/api/') && !path.startsWith('/api/public/')) {
    const token = req.headers.get('authorization')?.replace('Bearer ', '')

    if (!token) {
      return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
    }

    try {
      const user = await verifyJWT(token)
      // Inject player context into headers for downstream handlers
      const res = NextResponse.next()
      res.headers.set('x-grudge-id', user.uuid)
      res.headers.set('x-grudge-username', user.username)
      return res
    } catch {
      return NextResponse.json({ error: 'Invalid token' }, { status: 401 })
    }
  }
}

// Puter App — Full pattern for grudge-studio-app/index.html
// This runs INSIDE puter.com desktop environment
<script src="https://js.puter.com/v2/"></script>

class GrudgeApp {
  async init() {
    // 1. Auth — auto-triggers if not signed in
    if (!puter.auth.isSignedIn()) {
      await puter.auth.signIn()
    }
    const user = await puter.auth.getUser()

    // 2. Sync with Grudge backend (Grudge Backend (MySQL 8) + Crossmint wallet)
    const grudgePlayer = await this.syncWithBackend(user)

    // 3. Initialize player's cloud folder structure
    await this.initCloudFolders()

    // 4. Load cached game state from KV
    const cached = await puter.kv.get('grudge:playerState')
    this.playerState = cached ? JSON.parse(cached) : {}

    // 5. Set Puter window title + size
    puter.ui.setWindowTitle(`Grudge Studio — ${user.username}`)
    puter.ui.setWindowSize(1200, 800)

    return { user, grudgePlayer }
  }

  async initCloudFolders() {
    const dirs = [
      'grudge-studio',
      'grudge-studio/characters',
      'grudge-studio/islands',
      'grudge-studio/screenshots',
      'grudge-studio/saves',
      'grudge-studio/exports'
    ]
    for (const dir of dirs) {
      try {
        await puter.fs.mkdir(dir, { createMissingParents: true, failIfExists: false })
      } catch {} // already exists
    }
  }

  async saveCharacterArt(characterId, imageBlob) {
    // Save character image to player's Puter cloud
    await puter.fs.write(
      `grudge-studio/characters/${characterId}.png`,
      imageBlob,
      { overwrite: true }
    )
    // Get a public URL for the image (for NFT metadata)
    return await puter.fs.getReadURL(`grudge-studio/characters/${characterId}.png`)
  }
}
💡 Key pattern: Use puter.fs for large files (character images, island maps, save files). Use puter.kv for small frequently-read data (player prefs, cached balance, session state). Use your Grudge Backend (MySQL 8) API for all game-critical state.
Docker Backend Services

WORKER ARCHITECTURE

Docker services are containerized API services that run server-side. Use them as the bridge between your Puter frontend and your Grudge Backend (MySQL 8)/Crossmint backend.

grudge-id (Docker)

Account Sync

Puter auth → create/fetch Grudge Backend (MySQL 8) player → provision Crossmint wallet. Called on every sign-in.

game-api (Docker)

Game Logic

GBux awards, quest completions, character level-ups. All state mutations go through here, never from client directly.

wallet-service (Docker)

NFT Minting

Receives mint requests, calls Crossmint API, polls status, fires confirmation events back.

// grudge-id (Docker) — Full implementation
// Deploy to Puter as a serverless worker
import { router } from 'puter-workers'

// POST /api/sync — Called from frontend after puter.auth.signIn()
router.post('/api/sync', async ({ request, user }) => {
  if (!user?.puter) {
    return new Response(JSON.stringify({ error: 'Auth required' }), { status: 401 })
  }

  const puterUser = await user.puter.auth.getUser()

  // Check if player exists in our Grudge Backend (MySQL 8)
  const existingRes = await fetch(
    `${Grudge Backend (MySQL 8)_URL}/rest/v1/players?puter_id=eq.${puterUser.uuid}&select=*`,
    { headers: { 'apikey': Grudge Backend (MySQL 8)_ANON_KEY, 'Authorization': `Bearer ${Grudge Backend (MySQL 8)_SERVICE_KEY}` } }
  )
  const existing = await existingRes.json()

  if (existing.length > 0) {
    // Returning player — update last_seen, return profile
    return { player: existing[0], isNew: false }
  }

  // New player — create everything
  const [playerRes, walletAddress] = await Promise.all([
    // Grudge Backend (MySQL 8) insert
    fetch(`${Grudge Backend (MySQL 8)_URL}/rest/v1/players`, {
      method: 'POST',
      headers: {
        'apikey': Grudge Backend (MySQL 8)_ANON_KEY,
        'Authorization': `Bearer ${Grudge Backend (MySQL 8)_SERVICE_KEY}`,
        'Content-Type': 'application/json',
        'Prefer': 'return=representation'
      },
      body: JSON.stringify({
        puter_id: puterUser.uuid,
        username: puterUser.username,
        email: puterUser.email,
        gbux_balance: 100
      })
    }).then(r => r.json()),
    // Crossmint wallet
    provisionCrossmintWallet(puterUser.uuid, puterUser.email)
  ])

  // Store wallet + init KV
  await me.puter.kv.set(`player:${puterUser.uuid}:wallet`, walletAddress)

  return {
    player: { ...playerRes[0], wallet_address: walletAddress },
    isNew: true
  }
})

// GET /api/player/:puterUuid — Get player profile
router.get('/api/player/:id', async ({ params, user }) => {
  const res = await fetch(
    `${Grudge Backend (MySQL 8)_URL}/rest/v1/players?puter_id=eq.${params.id}&select=id,username,gbux_balance,wallet_address,created_at`,
    { headers: { 'apikey': Grudge Backend (MySQL 8)_ANON_KEY } }
  )
  return res.json()
})

Rendering Standards

RENDERING & EFFECTS SYSTEM

Consistent visual language, animation patterns, and performance standards shared across all Grudge Studio surfaces.

Design Tokens

Single Source of Truth

All colors, fonts, spacing, and animation values live in shared/design-tokens.js. Every app imports from there. Never hardcode a color or animation duration.

Animation System

Framer Motion Everywhere

Use Framer Motion for all React animations. Define reusable variants in the shared UI package. Stagger-in on page load, spring physics on interactions.

3D / Canvas

Three.js for Game World

Three.js r128 for island 3D previews and character viewers. Wrap in a React component in @grudge/ui. Use R2-cached assets, never inline large 3D files.

Performance

Core Web Vitals Rules

LCP under 2.5s (preload hero images). No layout shift (reserve space for dynamic content). First Input Delay under 100ms (defer non-critical JS).
