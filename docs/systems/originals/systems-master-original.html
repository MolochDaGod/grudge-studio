☁️ Puter Apps / Sites / Workers

How each Puter product type should be used and what they should share. Apps, Sites, and Workers each have distinct roles — don't mix them up.

PUTER APPS — Full Game Interfaces
Primary UI Layer

Puter Apps run inside the Puter desktop environment. They get the full Puter API: auth, storage, AI, KV, workers. Use these for full game screens — character creator, island view, inventory, marketplace.

Best Use

Grudge Game Hub App

The main game interface. Full access to puter.fs (player cloud), puter.kv (local prefs/cache), puter.ai (in-game AI), puter.workers (calls to your API). Registered once in Puter, accessible everywhere.

Standard Pattern

Every Puter App Must:

1. Import @grudge/auth and verify session on load
2. Sync with Vercel backend on first render
3. Use @grudge/ui design tokens for all styling
4. Store preferences in puter.kv with grudge: prefix

// Standard Puter App bootstrap — grudge-app/index.js
// Every Puter App starts with this pattern

const GrudgeApp = {
  async init() {
    // 1. Ensure Puter auth
    if (!puter.auth.isSignedIn()) {
      await puter.auth.signIn()
    }

    // 2. Get Puter identity
    const user = await puter.auth.getUser()

    // 3. Sync with Grudge backend
    const session = await fetch(
      `https://grudgegame.com/api/auth/sync`, {
        method: 'POST',
        body: JSON.stringify({ puterId: user.uuid, username: user.username })
      }
    ).then(r => r.json())

    // 4. Cache session in Puter KV
    await puter.kv.set('grudge:session', JSON.stringify(session))

    // 5. Load app with session context
    await renderApp(session)
  }
}

document.addEventListener('DOMContentLoaded', () => GrudgeApp.init())

PUTER WORKERS — Serverless API + Background Jobs
API Layer

Workers are the most powerful part of Puter for Grudge. They run server-side code, handle HTTP routes, and can access both the developer's Puter storage (me.puter) AND the user's storage (user.puter). Use them for lightweight API calls that don't need Supabase.

Worker: me.puter context

Studio-Owned Data

Leaderboards, shared game config, server-side analytics, cached game data. Stored in your Puter account. Players can't modify it.

Worker: user.puter context

Player-Owned Data

Character images, save files, screenshots, personal settings. Stored in each player's Puter cloud. Users pay for their own storage costs.

// Puter Worker: grudge-api-worker
// Handles player requests with full routing

// GET /api/player/profile
router.get('/api/player/profile', async ({ user }) => {
  if (!user) return { status: 401, error: 'Auth required' }

  // Read from user's own Puter KV (preferences)
  const prefs = await user.puter.kv.get('grudge:preferences')

  // Read from studio-owned KV (player stats cache)
  const stats = await me.puter.kv.get(`player_stats:${user.uuid}`)

  return { prefs, stats, username: user.username }
})

// POST /api/game/save
router.post('/api/game/save', async ({ request, user }) => {
  if (!user) return { status: 401 }
  const saveData = await request.json()

  // Save to user's Puter cloud (they own it, they pay)
  await user.puter.fs.write(
    `grudge-studio/saves/autosave.json`,
    JSON.stringify(saveData)
  )

  return { saved: true }
})

🔑 Shared Auth System — @grudge/auth

One auth package, consumed by every app. Puter identity → Grudge session → available everywhere with one hook.

// packages/auth/index.ts — @grudge/auth
// This single package handles auth for ALL Grudge properties

// ─── Types ──────────────────────────────────
export interface GrudgeSession {
  puterId:       string
  playerId:      string
  username:      string
  walletAddress: string
  gbuxBalance:   number
  isNewPlayer:   boolean
}

// ─── React Hook (Next.js + Puter Apps) ──────
export function useGrudgeAuth() {
  const [session, setSession] = useState<GrudgeSession | null>(null)
  const [loading, setLoading] = useState(true)

  useEffect(async () => {
    // Check Puter first (fastest)
    if (!puter.auth.isSignedIn()) { setLoading(false); return }

    // Check cached Grudge session
    const cached = await puter.kv.get('grudge:session')
    if (cached) { setSession(JSON.parse(cached)); setLoading(false); return }

    // Fresh sync with backend
    const user = await puter.auth.getUser()
    const grudge = await fetch(`${GRUDGE_API}/api/auth/sync`, {
      method: 'POST',
      body: JSON.stringify({ puterId: user.uuid, username: user.username })
    }).then(r => r.json())

    await puter.kv.set('grudge:session', JSON.stringify(grudge))
    setSession(grudge)
    setLoading(false)
  }, [])

  const signIn = async () => { await puter.auth.signIn(); window.location.reload() }
  const signOut = async () => {
    await puter.kv.del('grudge:session')
    await puter.auth.signOut()
  }

  return { session, loading, signIn, signOut, isAuthed: !!session }
}

// packages/api/gbux.ts — Shared GBux utilities
// Used by BOTH Vercel API routes AND Puter Workers

import { supabase } from './supabase'

export const gbux = {
  async award(playerId: string, amount: number, reason: string) {
    return supabase.rpc('award_gbux', { p_player_id: playerId, p_amount: amount, p_reason: reason })
  },
  async spend(playerId: string, amount: number, reason: string) {
    return supabase.rpc('spend_gbux', { p_player_id: playerId, p_amount: amount, p_reason: reason })
  },
  async balance(playerId: string) {
    const { data } = await supabase
      .from('players').select('gbux_balance').eq('id', playerId).single()
    return data?.gbux_balance ?? 0
  }
}

// packages/api/crossmint.ts — Shared Crossmint utilities

const CM_BASE = process.env.NODE_ENV === 'production'
  ? 'https://www.crossmint.com'
  : 'https://staging.crossmint.com'

export const crossmint = {
  async provisionWallet(email: string) {
    const res = await fetch(`${CM_BASE}/api/2022-06-09/wallets`, {
      method: 'POST',
      headers: { 'X-API-KEY': process.env.CROSSMINT_SERVER_KEY! },
      body: JSON.stringify({ type: 'solana-custodial-wallet', linkedUser: `email:${email}` })
    })
    return (await res.json()).publicKey
  },

  async mintCharacter(walletAddress: string, metadata: object) {
    return fetch(`${CM_BASE}/api/2022-06-09/collections/${process.env.CROSSMINT_CHARACTER_COLLECTION_ID}/nfts`, {
      method: 'POST',
      headers: { 'X-API-KEY': process.env.CROSSMINT_SERVER_KEY! },
      body: JSON.stringify({ recipient: `solana:${walletAddress}`, metadata, compressed: true })
    }).then(r => r.json())
  },

  async verifyWebhook(body: string, signature: string) {
    // HMAC verification — always verify before trusting mint events
    const hmac = crypto.createHmac('sha256', process.env.CROSSMINT_WEBHOOK_SECRET!)
    return hmac.update(body).digest('hex') === signature
  }
}
